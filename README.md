# 設備点検管理システム

施設管理の現場を想定して作成した、設備情報・点検履歴・点検期限を管理するWebアプリケーションです。
設備の登録、編集、削除、検索、点検履歴の登録、状態管理、ダッシュボード表示、CSV出力を行うことができます。

## 作成目的

施設管理・保全管理を行う業務システムを想定し、設備情報や点検履歴を管理できるWebアプリケーションを作成しました。
C#、ASP.NET Core MVC、SQL Server、HTML/CSS、JavaScriptを使用し、業務システム開発の基本的な流れを意識した構成にしています。
単なるCRUDアプリではなく、点検履歴の登録に応じて設備状態を更新したり、点検期限をもとに対応が必要な設備を把握できるようにしたりすることで、実際の施設管理業務に近い流れを意識しました。

* 設備一覧表示

* 登録設備数の表示
* 状態別件数の表示

  * 要対応
  * 故障

* 7日以内の点検予定件数の表示
* 注意が必要な設備一覧の表示

* 設備一覧のCSV出力

* 削除前の確認メッセージ表示
* 誤操作による削除を防ぐため、削除処理の前に確認ダイアログを表示

## 工夫した点

### 1. 業務を想定した設備状態の管理

設備状態を「正常」「要対応」「故障」「対応中」に分け、色付きラベルで表示しました。
文字だけでなく色でも状態を判断できるようにし、対応が必要な設備を見つけやすくしています。

### 2. 点検期限の可視化

次回点検日をもとに、期限超過や7日以内の点検予定を判定し、一覧画面やダッシュボードで強調表示しています。
施設管理の現場で、優先的に確認すべき設備を把握しやすくすることを意識しました。

### 3. 設備と点検履歴の1対多管理

設備情報と点検履歴を1対多で紐づけ、設備ごとに過去の点検結果を確認できるようにしました。
これにより、現在の状態だけでなく、過去の点検履歴も追える構成にしています。

### 4. 点検履歴登録時の設備状態更新

点検履歴を登録した際に、点検結果に応じて設備の状態を更新するようにしました。
例えば、点検結果が「故障」の場合、設備状態も「故障」に更新されます。

### 5. LINQによる検索・絞り込み

一覧画面では、検索条件をController側で受け取り、LINQを使ってSQL Serverから条件に合う設備を取得しています。
キーワード検索、状態絞り込み、点検期限絞り込みを組み合わせて検索できるようにしています。

### 6. CSV出力機能

設備一覧をCSV形式で出力できるようにしました。
検索条件を指定している場合は、その条件に一致した設備だけを出力できるため、必要な情報だけを取り出して確認できます。
また、CSV内にカンマや改行が含まれる場合でも形式が崩れないよう、エスケープ処理を実装しています。

## 起動方法

### 1. リポジトリをクローン

```bash
git clone https://github.com/ryuseitakagi0919/FacilityMaintenanceApp.git
```

### 2. プロジェクトフォルダへ移動

```bash
cd FacilityMaintenanceApp
```

### 3. パッケージを復元

```bash
dotnet restore
```

### 4. データベースを作成

```bash
dotnet ef database update
```

### 5. アプリを起動

```bash
dotnet run
```

起動後、表示されたURLにアクセスします。

```text
https://localhost:xxxx
```

## 学習・実装で意識したこと

このアプリでは、ASP.NET Core MVCの基本構成を意識して実装しました。

* Modelで設備情報や点検履歴のデータ構造を定義
* ControllerでDB操作や検索条件の処理を実装
* ViewでModelのデータを表示
* ViewModelでダッシュボード表示用の集計データを管理
* Entity Framework Coreを使用してSQL Serverと接続
* LINQを使用して検索・絞り込み処理を実装

業務システムとして、利用者が情報を確認しやすく、対応が必要な設備を見つけやすい画面構成を意識しました。
